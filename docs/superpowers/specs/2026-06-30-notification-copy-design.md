# 推送消息文案优化设计

## 目标

优化当前项目的推送消息标题和内容，让用户第一眼能判断消息含义：任务已经完成，或者正在等待权限批准。

## 范围

本次只调整 `plugin/watch-notify.js` 中默认通知标题和正文生成逻辑，并同步更新测试。配置格式、推送渠道、去重机制、事件监听机制都不改变。

## 文案策略

采用清晰直白风格。

任务完成标题：

```text
Opencode 任务完成：<会话标题>
```

如果会话标题为空或是默认新会话标题：

```text
Opencode 任务完成
```

任务完成内容：

```text
项目：<当前目录>
状态：任务已完成，可以查看结果
会话：<sessionID>
终端：<TTY，如有>
```

权限申请标题：

```text
Opencode 需要你批准操作
```

权限申请内容：

```text
项目：<当前目录>
权限：<权限类型>
操作：<匹配规则或标题>
会话：<sessionID>
```

## 实现方式

新增小型格式化函数集中生成任务完成标题、任务完成正文、权限申请正文。事件处理仍然只负责收集事件字段并调用分发逻辑。

## 兼容性

不修改 `watch-notify.json` 格式，不修改占位符变量名，不改变 Gotify、桌面通知、Webhook、SMTP、自定义命令的启用方式。

## 测试

在现有测试基础上增加对默认命令输出的可观察断言，验证任务完成和权限申请的标题、正文包含新的中文字段。
